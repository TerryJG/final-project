Outline

For CSD 1100, HTML & JavaScript, my final project is a personal portfolio website, where I showcase my real creative works in graphic design and video editing. This website will demonstrate my overall abilities in creating HTML structures, styling with CSS, and adding interactivity with JavaScript learnt throughout the course as well as using additional resources from W3Schools and YouTube.

Website Structure

In learning how to work with industry-standard frameworks such as React.js, my folder structure of my project is modeled after a React project. The "public" folder consists of all static content such as images and videos, as well as icons for buttons. The "src" folder, abbrivated from "source", includes all the CSS and JS files used for the portfolio website.

The website will feature five pages: Homepage, Projects, About Me, Contact, and Credits, with each page serving a specific purpose within the site's framework. All pages, except the landing page, will include a navigation bar positioned at the top, containing buttons linked to various sections. Additionally, a minimal site navigation bar will be present at the bottom of every page, visually showing the user the current location within the site.

The Homepage will be the landing page when a user visits the site. It is styled as a set of columns with featured images and videos. Near the top of the landing page will be a set of buttons acting as navigation links to different pages of the website using the <a href=""> element.

The Projects page is designed similarly to the Homepage, showcasing all the images and videos incorporated into the portfolio website. At the top of the Projects page, users will find a group of categories presented as buttons, created using the <button> element. Each category contains a selection of images and videos. When a user clicks on one of these category buttons, the JavaScript logic will filter out any images and videos not belonging to that category. For example, there are two buttons labeled "Graphic Design" and "Video Editing". If you click on "Graphic Design", the JavaScript will scan all the elements within the <main> container holding the images and videos. It will hide any elements that do not have a class called "image-item", which will leave only those that do.

The About Me page is a text-based resume, utilizing a flex-column layout to organize information under sections like Education, Experience, Skills, and Outreach. A side navigation bar is included exclusively in this page that allows users to navigate between sections using the <a href=""> element.

The Contact page is a form box featuring input fields for First Name, Last Name, Email, and Message, allowing users to reach out and connect.

Lastly, the Credits page will outline all my resources as well as my outline for the website.

Styling

Throughout the website's design process, my primary focus was on modularity. This meant that CSS files can be reused across the project and reduce repetition in code. Files such as the preloader.css, scrollbar.css, fonts.css, and column layouts are styled using this approach, which makes the overall look more consistent across pages. Additionally, this gives the added benefit of modifying a particular CSS file, and the changes can automatically be reflected throughout the website.

For styling, the portfolio website uses a mix of Flexbox and Tables using plain CSS. Flexbox ensures a consistent look and reliably arrange elements in either rows or columns. Tables provide a structured format for organizing and aligning information, used in the About Me page.

Each page is fully responsive and designed for multiple screen resolutions using Media Queries. Although CSS frameworks like Bootstrap or TailwindCSS could make website responsiveness more efficient, I wanted to both learn and demonstrate how responsiveness works without any abstractions.
